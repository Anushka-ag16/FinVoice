1.1  Vision
FinVoice is a three-tier AI-powered financial advisor — built for the complete beginner, intermediate investor, and seasoned trader alike. It combines Modern Portfolio Theory, advanced ML predictions, voice interaction, and explainable AI to deliver institutional-grade portfolio management at zero advisory cost for the free tier.

1.2  Investor Segments & Adaptive Questionnaire
The onboarding questionnaire dynamically adapts based on detected knowledge level. It asks progressively deeper questions and classifies users into one of three segments:


Adaptive Questionnaire Flow
Stage 1 (Universal): Age, income range, investment goal (wealth growth / retirement / education / emergency), time horizon
Stage 2 (Behavioral): Simulated loss scenario — 'If your portfolio dropped 20% tomorrow, you would...' (options reveal emotional bias)
Stage 3 (Conditional on answers): Experienced path asks about derivatives, sector bets, leverage; Beginner path asks about savings habits only
Stage 4 (Portfolio Input): Optional — input current holdings for instant analysis
Output: Risk Score (0-100), Investor Type label, Behavioral Bias tag (Loss Averse / Overconfident / Balanced)

IMPORTANT: Risk score is never static. It is recomputed after every major market event + on a 90-day questionnaire refresh cycle. Behavioral signals from app usage (panic sell attempts, frequent checking) also adjust the score dynamically.

3.1  Portfolio Optimization Layer
3.1.1  Base: Modern Portfolio Theory (MPT) + Extensions
PyPortfolioOpt — efficient frontier, max Sharpe, min volatility, Ledoit-Wolf covariance shrinkage
Black-Litterman Model — combine market equilibrium returns with your model's views (investor-specific)
Risk Parity — equal risk contribution per asset, better than equal-weight for downside protection
CVaR (Conditional Value at Risk) optimization — tail-risk aware, crucial for Indian market volatility

3.1.2  Return Prediction Models (This is the ML core)
These models predict expected returns per asset, which feed into the optimizer as 'views':


3.1.3  Ensemble & Signal Aggregation
Combine LSTM, TFT, XGBoost predictions using a meta-learner (Stacking Regressor)
Weight each model's signal by its recent Sharpe contribution (rolling 30-day evaluation)
Final output: Expected Return vector + Covariance Matrix → fed into PyPortfolioOpt CVaR optimizer
Confidence intervals from model ensemble determine position sizing (low confidence = smaller position)

3.2  Holdings Analyzer
Input: User's current portfolio (asset, quantity, buy price)
Exposure Engine: Classify each holding → sector, asset class, geography, market cap tier
Concentration Risk: Flag any single stock > 20% of portfolio, any sector > 35%
Correlation Analysis: Compute pairwise correlations from 1-year rolling window. Flag highly correlated pairs (r > 0.75) as redundant diversification
Beta Calculation: Portfolio beta vs Nifty 50. If beta > 1.3 for 'Moderate' risk user → alert
Rupee-Level Rebalancing: LP solver (scipy.optimize or PuLP) minimizes trades needed to reach target allocation while minimizing capital gains tax events

3.3  New Money Advisor
When user says 'I want to invest ₹1L', the flow is:

Input Collection: Maximum acceptable loss (worst-case), goal (growth/income/preservation), horizon, tax bracket
Existing Risk Profile loaded from DB — no re-questionnaire unless profile is stale (> 90 days)
ML Engine computes expected return / risk for each candidate asset based on current regime
Optimizer runs 3 scenarios: Conservative / Balanced / Aggressive — user sees all 3 with projected returns
Output: Exact rupee allocation — e.g., ₹40K Nifty50 ETF, ₹25K SGB, ₹20K PPFAS Flexi, ₹15K FD
Explainable AI block explains WHY each allocation was chosen (see Section 7)

3.4  Crash Simulation / Stress Testing
Monte Carlo Simulation: 10,000 portfolio paths using Cholesky decomposition of covariance matrix. Output: 5th/50th/95th percentile portfolio value at each time step
Historical Scenario Replay: Apply actual return series from 2008 Global Crisis, 2020 COVID crash, 2022 rate hike cycle to user's current portfolio
Custom Shocks: User can define 'what if Nifty drops 30%' — system computes portfolio impact using beta + correlation matrix
Output: Maximum Drawdown, Time to Recovery, Probability of Ruin (portfolio < 50% of initial)

3.5  Daily Portfolio Drift Detection
Every night at 11 PM IST: Pull latest prices, recompute portfolio allocation %
Compare actual allocation vs target allocation (from risk profile)
Drift Threshold: If any asset class deviates > 5% from target, generate rebalancing alert
Severity Levels: INFO (1-3% drift) → WARN (3-5% drift) → ALERT (>5% drift)
Push notification for paid users; weekly digest email for free users
This is the most important retention feature — it keeps users engaged and prevents the portfolio from silently drifting into a risk zone the user never agreed to.

9.3  Model Evaluation Metrics
IC (Information Coefficient): Spearman correlation between predicted and actual forward returns. Target: IC > 0.05 consistently
ICIR (IC Information Ratio): IC / std(IC). Target: ICIR > 0.5 (signal is stable, not noisy)
Sharpe Ratio of paper portfolio: Target > 1.0 on 2022-2024 out-of-sample
Maximum Drawdown on test period: Must be below user's stated risk tolerance
Calibration: Predicted probability of loss should match actual frequency (reliability diagram)

3.  ML / AI MODELS — PORTFOLIO ENGINE	

Model	Purpose & Details	
LSTM (Long Short-Term Memory)	Primary time-series predictor. Input: OHLCV + macro features (30-day window). Output: 1-day and 5-day return probability distribution. Train with PyTorch or TensorFlow. One model per asset class (not per stock — use embeddings).	
Temporal Fusion Transformer (TFT)	State-of-the-art for multi-horizon forecasting. Handles known future inputs (earnings dates, RBI meetings) + past observations. Use Darts library or PyTorch Forecasting.	
XGBoost / LightGBM	Tabular features: RSI, MACD, Bollinger Bands, PE ratio, promoter holding %, FII/DII flows. Faster to train, great for feature importance (used in Explainable AI).	
Prophet (Meta)	Trend + seasonality decomposition per stock. Useful for detecting structural breaks and adding seasonality features to other models.	
Random Forest Regressor	Ensemble baseline. Predicts relative return rank within sector — used for stock screening within a recommended category.	
GARCH / EGARCH	Volatility modeling — predict next-period volatility per asset. Used for dynamic position sizing and crash probability estimation.	
Gaussian Mixture Model (GMM)	Regime detection — classify current market into Bull / Bear / Sideways / High-Volatility. Allocation shifts based on regime. Scikit-learn.	
Sentiment NLP (FinBERT)	Fine-tuned BERT for financial text. Score news headlines and social media per stock. Hugging Face — free model weights. Add sentiment score as feature to LSTM/XGB.	
Correlation Graph (GCN)	Graph Convolutional Network on stock correlation matrix. Captures sector-level contagion risk — e.g., if HDFC drops, what else is at risk? PyTorch Geometric.

6.  EXPLAINABLE AI (XAI) LAYER	

Technique	Where Applied	
SHAP (SHapley Additive Explanations)	For XGBoost/LightGBM predictions — shows which features (e.g., RSI, FII flow, PE ratio) drove a stock's predicted return up or down. shap Python library.	
LIME (Local Interpretable Model-agnostic)	For LSTM predictions — perturbs input to understand which recent price events most influenced the forecast.	
Attention Visualization (TFT)	Temporal Fusion Transformer has built-in attention weights — shows which time steps were most important for the prediction.	
Factor Attribution	Decompose portfolio return into: market beta, sector tilt, individual stock alpha, and unexplained noise. Barra-style attribution.	
Decision Tree Surrogate	Train a shallow decision tree to mimic the ML model's decisions on a local neighborhood — produces if-then rules readable to users.	
Natural Language Generation (NLG)	LLM converts SHAP values + factor attribution into a paragraph: 'We increased your gold allocation because FII outflows and rising US yields suggest equity headwinds over the next 30 days.'
